# 指令资源与能力注册设计

本文档描述 Tavolo 当前的外置指令资源、能力描述和运行时注册模型。文档只记录当前设计，不记录实施过程。

## 设计目标

- 指令素材、字体和配置从 jar 中外置到 `assets/` 目录，降低核心 jar 体积。
- handler 和 generator 使用统一的英文 `id` 作为资源目录、配置文件和注册表描述的主键。
- HTTP 服务、客户端和外部调用方通过统一的 `CommandDescriptor` 获取能力信息，不直接依赖 KSP 生成细节。
- 运行时资源读取统一经过 `common` 模块的 `Assets`，避免业务代码直接拼接 classpath 或项目路径。
- Release 可通过 Gradle 任务把 `assets/` 打包为独立资源 zip。

## 目录结构

仓库内资源目录位于项目根目录 `assets/`：

```text
assets/
  version.yml
  handlers/
    round/
      handler.yml
      assets/
      fonts/
    ...
  generators/
    blue_archive/
      generator.yml
      assets/
      fonts/
    ...
```

`handlers/{id}` 和 `generators/{id}` 是资源包的最小更新单元。目录名必须与配置文件中的 `id` 一致。

## 资源目录解析

资源读取由 `common/src/main/kotlin/assets/Assets.kt` 提供：

```kotlin
data class AssetsConfig(
    val rootDir: Path = AssetsConfig.defaultRootDir()
)
```

默认解析顺序：

1. 系统属性 `tavolo.assets.dir`。
2. 环境变量 `TAVOLO_ASSETS_DIR`。
3. 当前工作目录下的 `assets`、`../assets`、`../../assets`。
4. 以上都不存在时使用当前工作目录下的 `assets`。

`Assets.resolve(path)` 要求传入相对路径，并校验解析结果不能越出资源根目录。

## 配置文件

### version.yml

资源根目录必须包含：

```yaml
version: "2.0.0"
time: "2026-05-09 00:00:00"
```

字段含义：

- `version`：资源包版本。
- `time`：资源包构建或更新时间。

### handler.yml

handler 目录必须包含 `handler.yml`：

```yaml
id: round
type: kotlin
version: 1
name: "Round"
regex: "(?i)round"
```

字段含义：

- `id`：能力唯一 id，必须等于目录名。
- `type`：当前校验允许 `kotlin` 或 `template`；运行时已接入 `kotlin`，`template` 仍是保留格式。
- `version`：单个能力配置版本，缺省时按 `1` 处理。
- `name`：展示名。
- `regex`：外部指令匹配正则；缺省时回退到 `name`。

### generator.yml

generator 目录必须包含 `generator.yml`：

```yaml
id: blue_archive
version: 1
name: "BlueArchive"
regex: "BlueArchive"
fonts:
  - name: glow-sans-sc-normal-heavy
    file: fonts/GlowSansSC-Normal-Heavy.otf
```

generator 固定视为 Kotlin 实现，不允许声明 `type` 或 `layout`。

## 资源校验与打包

`core` 模块提供两个 Gradle 任务：

- `validateCommandAssets`：校验 `assets/version.yml`、handler/generator 配置、目录 id 一致性和配置引用的资源文件。
- `packageCommandAssets`：依赖 `validateCommandAssets`，把 `assets/` 打包为 `tavolo-assets-${version}.zip`。

Release 产物应同时提供 jar 和资源 zip。运行时只读取已解压的资源目录，不负责下载或解压 zip。

## 能力注册

`core/src/main/kotlin/registry/CommandRegistry.kt` 定义统一能力模型：

```kotlin
enum class CommandCategory { HANDLER, GENERATOR }
enum class CommandType { TEMPLATE, KOTLIN }

data class CommandDescriptor(
    val id: String,
    val category: CommandCategory,
    val type: CommandType,
    val name: String,
    val regex: Regex,
    val version: Int,
)
```

`CommandRegistry.load()` 从资源目录加载配置，并把配置与 KSP 生成的 `handlerMap`、`generatorMap` 合并：

- `type: kotlin` 的 handler 必须存在同 id 的 KSP 注册项。
- generator 必须存在同 id 的 KSP 注册项。
- handler 和 generator 的 id 不能重复。
- `CommandRegistry.descriptors` 为 HTTP 服务和外部调用方提供统一能力列表。

## 当前限制

- `template` handler 的配置校验已预留，运行时执行链路尚未接入。
- 配置文件解析当前使用轻量 key-value 解析，仅覆盖当前资源配置需要的字段。
- 资源包下载、缓存、增量更新不在运行时处理，交由部署流程或外部系统完成。
