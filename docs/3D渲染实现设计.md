# 3D 渲染实现设计

本文档描述 `graphics` 模块中 `top.e404.tavolo.draw.render3d` 包的当前实现。该渲染器是一个基于 CPU 的轻量 3D 光栅化器，主要用于生成简单几何体、UV 方块模型和静态预览图。

## 设计目标

- 提供不依赖 GPU 的 3D 渲染能力，输出 Skia `Image`。
- 支持基础 Mesh、轨道相机、透视/正交投影、实体/线框渲染、简单光照和阴影贴图。
- 让核心光栅化逻辑可以脱离 Skia `Bitmap` 做单元测试，通过 `RenderTarget` 记录绘制指令。
- 保持现有 `renderSceneToImage(scene, config)` 使用方式稳定。

## 模块结构

- `struct.kt`：基础数学和数据结构，包括 `Vec2`、`Vec3`、`Vec4`、`Mat4`、`Vertex`、`Face`、`Mesh`、`Transformation`。
- `mesh.kt`：长方体、UV 长方体和 Mesh 合并工具。
- `mesh_extras.kt`：平面 Mesh 构造。
- `light.kt`：简单 Lambert 光照和颜色应用。
- `shadow.kt`：`ShadowMap` 和 Blinn-Phong 光照。
- `target.kt`：渲染目标与纹理抽象，包括 Skia 适配器和记录式测试目标。
- `render.kt`：相机、渲染配置、阴影通道、主渲染通道和三角形光栅化。

## 核心数据模型

`Mesh` 由顶点列表、面索引列表和可选 Skia `Bitmap` 纹理组成：

```kotlin
data class Mesh(
    val vertices: List<Vertex>,
    val faces: List<Face>,
    val texture: Bitmap? = null,
)
```

`Face.indices` 支持三角形和多边形。渲染时使用 fan triangulation，将一个面拆成 `(0, i + 1, i + 2)` 形式的三角形序列。

`OrbitCamera` 使用目标点、yaw、pitch 和 distance 计算相机世界坐标、视图矩阵和前向向量。`ViewMatrix.cameraPosition` 会在主渲染通道中用于高光方向和背面剔除判断。

## 渲染目标抽象

3D 光栅化不是命令式布局绘制，而是逐像素写入颜色和深度。因此当前抽象没有复用 Compose 的 `DrawCanvas`，而是拆成更贴近光栅化的接口：

```kotlin
interface RenderTexture {
    val width: Int
    val height: Int
    fun getColor(x: Int, y: Int): Int
}

interface RenderTarget : RenderTexture {
    fun clear(color: Int)
    fun setPixel(x: Int, y: Int, color: Int)
    fun drawLine(x0: Float, y0: Float, x1: Float, y1: Float, color: Int)
}
```

生产路径使用 `BitmapRenderTarget` 和 `BitmapRenderTexture` 适配 Skia `Bitmap`。测试路径可以使用 `RecordingRenderTarget` 记录 `Clear`、`Pixel`、`Line` 指令，断言实体模式是否写像素、线框模式是否画边线。

当前 `Mesh.texture` 仍保留为 `Bitmap?`，避免破坏已有公共 API。内部进入主通道时会包装成 `BitmapRenderTexture`。`renderMeshMainPass` 和 `rasterizeTriangleMain` 也保留了旧的 `Bitmap` 参数重载，兼容直接调用这些函数的代码。

## 渲染管线

`renderSceneToImage` 是主要入口，流程如下：

1. 根据 `RenderConfig` 计算超采样尺寸。
2. 通过 `OrbitCamera.createViewMatrix()` 得到视图矩阵和相机位置。
3. 根据 `usePerspective` 选择透视投影或正交投影。
4. 构造光源视角的正交投影视图矩阵。
5. 执行阴影通道，生成 `ShadowMap`。
6. 执行主渲染通道，写入超采样 `RenderTarget`。
7. 根据 `antiAliasingLevel` 做平均下采样。
8. 返回最终 Skia `Image`。

## 阴影通道

`renderShadowPass` 从光源视角遍历所有 Mesh 面：

- 跳过顶点数小于 3 的无效面。
- 将多边形按 fan triangulation 拆成三角形。
- 使用 `lightVP` 将三角形顶点变换到光源裁剪空间。
- 如果三个顶点都在光源裁剪空间背后，直接跳过。
- 调用 `ndcToScreen` 转为阴影贴图坐标。
- 调用 `rasterizeTriangleShadow` 写入最近深度。

`ShadowMap` 使用 `FloatArray` 存储深度，初始值为 `Float.POSITIVE_INFINITY`。写入时只保留更小的深度值。

## 主渲染通道

`renderMeshMainPass` 负责处理 Mesh 的面、法线、背面剔除和实体/线框模式：

- 面法线使用前三个顶点计算，当前是 flat shading。
- `useBackFaceCulling` 开启时，根据面法线和相机方向跳过部分面。
- `renderFaces = true` 时进入实体光栅化。
- `renderFaces = false` 时调用 `RenderTarget.drawLine` 绘制三角形三条边。

实体光栅化由 `rasterizeTriangleMain` 完成：

- 计算三角形屏幕包围盒。
- 使用重心坐标判断像素是否在三角形内。
- 使用 `oneOverW` 做透视校正的深度、世界坐标和 UV 插值。
- 通过 zBuffer 保留最近像素。
- 使用 `lightVP` 查找阴影贴图，得到 `shadowFactor`。
- 如果 Mesh 有纹理，则采样纹理；透明度小于 200 的 texel 会跳过，不写颜色和深度。
- 使用 Blinn-Phong 计算最终颜色。
- 调用 `RenderTarget.setPixel` 写入目标。

## 光照模型

当前有两套光照辅助：

- `calculateLightIntensity` / `applyLight`：简单 Lambert 光照，主要用于线框颜色。
- `calculateBlinnPhong`：主实体渲染使用，包含环境光、漫反射、高光和阴影因子。

阴影只影响漫反射和高光，环境光始终保留，避免阴影区域完全变黑。

## 抗锯齿

`RenderConfig.antiAliasingLevel` 使用超采样实现：

- 主渲染目标尺寸为 `width * aaLevel` 和 `height * aaLevel`。
- `aaLevel > 1` 时，对每个输出像素对应的超采样像素做 ARGB 平均。
- `aaLevel == 1` 时逐像素复制超采样目标到最终目标。

该实现简单稳定，但逐像素循环成本较高。

## 测试策略

当前测试分三类：

- 纯逻辑测试：覆盖向量、矩阵、相机、Mesh 构造、光照和 `ShadowMap`。
- 小尺寸像素测试：使用真实 `BitmapRenderTarget`，断言像素颜色、zBuffer、透明纹理跳过、阴影明暗和端到端渲染结果。
- 指令记录测试：使用 `RecordingRenderTarget`，断言实体模式产生 `Pixel` 指令，线框模式产生 `Line` 指令。

这种组合可以同时验证数学逻辑、真实 Skia 写入路径和可 mock 的光栅化输出行为。

## 已知限制

- 当前没有做完整视锥裁剪，三角形跨越裁剪面时可能出现异常投影。
- 背面剔除规则仍较简化，复杂模型和不同坐标约定下可能需要进一步校准。
- 法线是按面计算的 flat shading，不支持顶点法线和平滑着色。
- 纹理只支持最近邻采样，没有 mipmap、双线性过滤或 wrap mode。
- 阴影贴图没有 PCF 过滤，阴影边缘会偏硬。
- `Mesh.texture` 仍绑定 Skia `Bitmap`，当前只在内部通过 `RenderTexture` 适配；如果未来需要完全脱离 Skia，可进一步把 Mesh 纹理类型改为 `RenderTexture?` 或提供新的 Mesh 类型。
- CPU 光栅化适合静态图和小规模预览，不适合大量高分辨率实时渲染。
