# 人工测试说明

人工测试用于生成图片或动图，结果需要开发者人工查看。可自动断言的用例应优先放到 `src/test`。

## 命名规则

- 人工测试类统一使用 `*ManualTest` 后缀。
- 公共辅助类使用 `*ManualTestSupport` 后缀，避免被误认为普通业务类。
- 文件名与主要测试类名保持一致；一个文件内可包含少量强相关的人工测试类。

## 输出目录

人工测试工作目录为仓库根目录下的 `run`。

core 模块的 handler 和 generator 输出统一写入：

```text
run/out/manual/core
```

文件名使用类型和输入区分：

- `generator-<GeneratorName>.png`
- `generator-<GeneratorName>.gif`
- `handler-<HandlerName>-<InputName>-<InputExt>.png`
- `handler-<HandlerName>-<InputName>-<InputExt>.gif`

graphics 模块继续按功能输出到 `run/out/compose`、`run/out/render3d` 等目录。

## 运行方式

运行全部人工测试：

```shell
./gradlew manualTest
```

运行单个模块：

```shell
./gradlew :core:manualTest
./gradlew :graphics:manualTest
```

运行单个类：

```shell
./gradlew :core:manualTest --tests "*BasicImageHandlerManualTest"
./gradlew :graphics:manualTest --tests "*ComposeThemeManualTest"
```

项目默认开启并行执行，配置位于仓库根目录的 `gradle.properties`：

```properties
manualTest.parallel=true
manualTest.maxParallelForks=4
```

临时关闭并行：

```shell
./gradlew manualTest -PmanualTest.parallel=false
```

指定并行 fork 数：

```shell
./gradlew manualTest -PmanualTest.parallel=true -PmanualTest.maxParallelForks=4
```

并行开启后，测试不得清空共享输出目录，不同测试应通过文件名区分输出。
